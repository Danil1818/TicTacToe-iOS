# TicTacToe для iPhone через Xcode

Native iOS-приложение «Крестики-нолики» для iPhone. Проект должен быть создан в Xcode и реализован на SwiftUI: современный интерфейс, игровое поле 3x3, режим для двух игроков, режим против компьютера, проверка победы, поражения, ничьей и кнопка `Restart`.

## Цель проекта

Создать полноценное iPhone-приложение в Xcode, которое можно запускать на iPhone.

## Технологии

- Xcode
- Swift
- SwiftUI
- iOS Simulator
- MVVM-подход для разделения UI и игровой логики

## План разработки

1. Создать новый Xcode project.
   - Template: `iOS App`
   - Interface: `SwiftUI`
   - Language: `Swift`
   - Product Name: `TicTacToe`

2. Настроить структуру проекта.
   - `TicTacToeApp.swift` — точка входа приложения.
   - `ContentView.swift` — основной экран игры.
   - `GameViewModel.swift` — состояние и логика игры.
   - `GameMode.swift` — режимы игры.
   - `GameResult.swift` — результат партии.
   - `Player.swift` — игроки `X` и `O`.

3. Разработать интерфейс.
   - Заголовок игры.
   - Переключатель режимов: `2 игрока` и `Против компьютера`.
   - Табло счета: игрок X, игрок O или компьютер, ничьи.
   - Статус текущего хода.
   - Игровое поле 3x3.
   - Финальный блок с результатом.
   - Кнопка `Restart`.

4. Реализовать основной игровой процесс.
   - Первый ход всегда делает `X`.
   - Игроки ходят по очереди.
   - Нельзя выбрать уже занятую клетку.
   - После каждого хода приложение проверяет результат.
   - После окончания партии поле блокируется.

5. Реализовать все варианты победы.

```swift
let winningCombinations = [
    [0, 1, 2],
    [3, 4, 5],
    [6, 7, 8],
    [0, 3, 6],
    [1, 4, 7],
    [2, 5, 8],
    [0, 4, 8],
    [2, 4, 6]
]
```

Схема индексов:

```text
0 | 1 | 2
3 | 4 | 5
6 | 7 | 8
```

6. Реализовать ничью.
   - Если все 9 клеток заполнены.
   - Если победной комбинации нет.
   - Показать сообщение `Ничья`.

7. Реализовать режим против компьютера.
   - Человек играет за `X`.
   - Компьютер играет за `O`.
   - После хода человека компьютер делает ход автоматически.
   - На время хода компьютера поле блокируется.

8. Реализовать алгоритм компьютера.
   - Сначала попытаться выиграть.
   - Если игрок может выиграть следующим ходом, заблокировать его.
   - Если центр свободен, занять центр.
   - Если свободен угол, занять угол.
   - Если остались боковые клетки, выбрать первую свободную.

9. Реализовать финальные сообщения.
   - Если игрок победил: `Ты выиграл, молодец!`
   - Если компьютер победил: `Ты проиграл`
   - Если игра закончилась без победителя: `Ничья`

10. Реализовать кнопку `Restart`.
    - Очистить поле.
    - Сбросить текущий ход на `X`.
    - Скрыть финальное сообщение.
    - Оставить счетчик партий.

11. Проверить приложение.
    - iPhone SE simulator.
    - iPhone 15 simulator.
    - Светлая и темная тема.
    - Победа X.
    - Победа O.
    - Победа компьютера.
    - Блокировка победы игрока компьютером.
    - Ничья.
    - Restart после каждого финального результата.

## Основные тезисы игры

- Должны быть обработаны победа, поражение и ничья.
- После окончания партии пользователь должен видеть понятный результат.
- Пользователь всегда должен иметь возможность начать новую партию через `Restart`.
